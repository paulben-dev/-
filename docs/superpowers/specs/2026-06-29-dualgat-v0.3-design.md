# DualGAT v0.3 — Design Spec

**Date:** 2026-06-29
**Based on:** DualGAT paper (arXiv:2504.10078v2) — Dual Graph Attention Network
**Prerequisite:** v0.1 MVP + v0.2 MS-LSTM

---

## Overview

Add a Dual Graph Attention Network on top of pre-trained MS-LSTM to propagate sparse expert signals across related stocks via two graph structures (industry + correlation). The 2-hop GAT with dual-graph attentive fusion refines predictions beyond what MS-LSTM alone achieves.

**Success criterion:** DualGAT achieves higher mean IC than both rule-based baseline and MS-LSTM on the same backtest period.

---

## Architecture

```
Input: 3 features per stock (MS-LSTM pred, expert_available, expert_signal)
  │
  ├─→ Industry Graph (GICS sector) ──→ GATConv(hop=1) ─→ H_ind ─┐
  │                                                               │
  ├─→ Correlation Graph (30d price) ─→ GATConv(hop=1) ─→ H_cor ─┤
  │                                                               │
  │    Dual-graph attentive fusion:                               │
  │      β_ind, β_cor = softmax(q·H)                              │
  │      H_fused = β_ind ⊙ H_ind + β_cor ⊙ H_cor                  │
  │      │                                                        │
  │      ├─→ GATConv(hop=2) ─→ second fusion ─→ H_final [N, 32]  │
  │      │                                                        │
  │      └─→ MLP: [32 → 1] → scalar prediction per stock          │
  │
  Output: [N] predicted return ratios
```

- **2-hop GAT**: each GAT layer followed by dual-graph attentive fusion with learnable β weights
- **Multi-head attention**: 4 heads per GATConv layer
- **Hidden dimensions**: d_hidden=64, d_out=32
- **Dropout**: 0.2 in GAT layers

---

## Graph Construction

### Industry Graph
- Fetch GICS sector for each stock via yfinance fundamentals
- Edge between stocks iff same GICS sector
- Static — built once, reused across all dates

### Correlation Graph
- 30 trading-day price correlation between stock pairs
- Edge if |correlation| > θ1 (0.77), or > θ2 (0.67) if either stock has expert label
- Dynamic — rebuilt per trading day

### Graph Format
- Both graphs stored as PyG `edge_index` tensors [2, num_edges]
- Self-loops included (node connects to itself)

---

## Data Flow

```
1. Load pre-trained MS-LSTM → generate base predictions per stock-day
2. ExpertTracker → expert_available + expert_signal per stock-day
3. Build input features [N_stocks, 3] per trading day
4. Build graphs (industry once, correlation per day)
5. Train DualGAT with cross-sectional IC Loss per day
6. Train/Val split: first 80% dates train, last 20% validate
```

- **MS-LSTM frozen**: not fine-tuned, feature extraction only
- **Batch**: one batch = all stocks on one trading day (cross-sectional)
- **Optimizer**: Adam, lr=1e-3, weight_decay=1e-5
- **Early stopping**: validation IC no improvement for 10 consecutive epochs
- **Device**: GPU if available, CPU otherwise

## Testing Strategy

1. **Graph building**: industry graph edges correct (same sector → connected), correlation graph respects thresholds
2. **Model structure**: forward() produces [N] output, handles both graph inputs
3. **IC Loss**: perfect prediction → loss≈0, random → loss≈1
4. **Predictor interface**: `predict()` returns DataFrame matching BaselinePredictor/MSLSTMPredictor format
5. **End-to-end**: 5 stocks, 30 days, 1 epoch, no errors
6. **Save/Load**: round-trip produces identical predictions

---

## Non-Goals

- Hyperparameter tuning / grid search
- Joint training of MS-LSTM + DualGAT (MS-LSTM frozen)
- Dashboard model-switching UI
- Real-time graph updates
- Multi-GPU or distributed training
