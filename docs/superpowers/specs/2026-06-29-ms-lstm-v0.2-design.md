# MS-LSTM Stock Prediction v0.2 — Design Spec

**Date:** 2026-06-29
**Based on:** DualGAT paper (arXiv:2504.10078v2) — MS-LSTM temporal pre-training
**Prerequisite:** v0.1 MVP (rule-based predictor, expert tracker, backtest engine)

---

## Overview

Replace the rule-based baseline predictor with a Multi-Scale LSTM model that captures temporal patterns at different time scales (1/2/4/8/16 day strides). Reuse all v0.1 infrastructure: expert tracker, backtest engine, and FastAPI dashboard.

**Success criterion:** MS-LSTM achieves higher mean IC than the rule-based baseline on the same backtest period.

---

## Architecture

```
Input: 30-day OHLCV + expert features per stock
  │
  ├─→ LSTM(stride=1)  ─┐
  ├─→ LSTM(stride=2)  ─┤
  ├─→ LSTM(stride=4)  ─┼─→ mean(last_hidden_states) ─→ MLP([64,32,1]) ─→ predicted_return
  ├─→ LSTM(stride=8)  ─┤
  ├─→ LSTM(stride=16) ─┘
  │
  └─→ Expert features: expert_available (0/1) + expert_signal (continuous)
         concatenated directly to MLP input
```

- **5 independent LSTM branches**, each sampling the 30-day window at a different stride
  - stride=1: 30 time steps, stride=2: 15 steps, stride=4: ~8 steps, stride=8: ~4 steps, stride=16: ~2 steps
- **Last hidden state** from each branch → mean pooling → concatenate expert features → 2-layer MLP → scalar prediction
- **IC Loss**: `1 - Pearson_correlation(predictions, actual_returns)` computed cross-sectionally per day
- **Hidden dimensions**: each LSTM hidden=64, MLP layers [64 → 32 → 1], dropout=0.2

---

## Data Flow & Training Pipeline

```
Training date range (e.g. 2024-01-01 ~ 2024-06-30)
  │
  ├─→ Sliding window: for each trading day, take prior 30 days OHLCV → (N_days, N_stocks, 30, features)
  ├─→ Labels: actual daily return ratio (close_today / close_yesterday - 1)
  ├─→ Expert features: ExpertTracker generates expert_available + expert_signal per stock-day
  │
  └─→ Train/Val split: first 80% dates train, last 20% validate
        IC Loss = 1 - Pearson_correlation(predictions, actual_returns) per batch
```

- **Batch**: one batch = all stocks on one trading day (cross-sectional IC loss)
- **Optimizer**: Adam, lr=1e-3, weight_decay=1e-5
- **Early stopping**: validation IC no improvement for 10 consecutive epochs
- **Device**: GPU if available, CPU otherwise

## Key Interfaces

```python
# src/model/ms_lstm.py

class MSLSTMModel(nn.Module):
    """Multi-Scale LSTM for stock return prediction."""
    def __init__(
        self,
        input_dim: int,           # OHLCV features per time step
        hidden_dim: int = 64,
        num_scales: int = 5,      # stride = 2^i for i in [0..4]
        expert_feat_dim: int = 2, # expert_available + expert_signal
        dropout: float = 0.2,
    )
    def forward(
        self,
        price_features: Tensor,   # [N_stocks, seq_len=30, input_dim]
        expert_features: Tensor,  # [N_stocks, expert_feat_dim]
    ) -> Tensor                   # [N_stocks] predicted returns


class MSLSTMPredictor:
    """Compatible with v0.1 BaselinePredictor interface."""
    def fit(
        self,
        stocks: list[str],
        start_date: str,
        end_date: str,
        expert_records_by_date: dict[str, list[ExpertRecord]] | None = None,
        epochs: int = 100,
        lr: float = 1e-3,
    ) -> dict  # training history (train_loss, val_ic per epoch)

    def predict(
        self,
        stocks: list[str],
        date_str: str,
        expert_records: list | None = None,
    ) -> pd.DataFrame  # columns: [stock, date, predicted_return, signal_source]

    def save(self, path: str | Path) -> None
    def load(self, path: str | Path) -> None
```

## Testing Strategy

1. **Model structure**: forward() produces correct output shape [N_stocks]
2. **Stride sampling**: each LSTM branch receives correct number of time steps
3. **IC Loss**: perfect predictions → loss ≈ 0, random predictions → loss ≈ 1
4. **Predictor interface**: `MSLSTMPredictor.predict()` returns DataFrame matching BaselinePredictor format
5. **End-to-end**: train on small dataset (5 stocks, 30 days), 1 epoch, no errors
6. **Save/Load**: round-trip model save and load produces identical predictions

---

## Non-Goals

- Hyperparameter tuning / grid search
- GPU requirement (CPU-compatible)
- Real-time / streaming training
- Dashboard model-switching UI (defer to v0.3)
- Paper's full StockTwits dataset replication
- Multi-GPU or distributed training
